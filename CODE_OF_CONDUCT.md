# Code of Conduct

## Our Pledge

We want Open Spaced Repetition to be a welcoming, respectful, and productive community for everyone.
Project participants are expected to treat each other with empathy, patience, and professionalism in issues, pull requests, discussions, reviews, and any other project space.

## Expected Behavior

Examples of behavior that contributes to a positive environment include:

- Using welcoming and inclusive language
- Respecting differing viewpoints and experiences
- Giving and accepting constructive feedback
- Staying focused on what is best for the project and its users
- Showing patience with newcomers and contributors who are learning the codebase

## Unacceptable Behavior

Examples of unacceptable behavior include:

- Harassment, intimidation, threats, or personal attacks
- Discriminatory language or conduct
- Sexualized language, imagery, or attention
- Trolling, insulting comments, or sustained disruption of project work
- Publishing private information without explicit permission
- Any other conduct that would reasonably be considered inappropriate in a professional open source community

## Scope

This Code of Conduct applies in all project spaces, including the GitHub repository, issues, pull requests, discussions, release channels, and any community spaces used for Open Spaced Repetition project work.

It also applies when an individual represents the project in public spaces, such as using an official project account, speaking on behalf of the project, or presenting project work at events.

## Reporting

If you experience or witness unacceptable behavior, please report it to the project maintainers.

When possible, include:

- A description of what happened
- Links, screenshots, or other relevant context
- The names or usernames of the people involved
- Whether the situation is ongoing or urgent

Reports will be handled with care and discretion. Maintainers will avoid sharing identifying information beyond what is necessary to investigate and respond.

## Enforcement

Maintainers are responsible for clarifying and enforcing this Code of Conduct. They may take action that they determine is appropriate, including:

- Asking a participant to stop specific behavior
- Removing comments, commits, code, issues, or pull requests
- Temporarily or permanently limiting participation in project spaces
- Reporting behavior to GitHub or other relevant platform administrators

Enforcement decisions should be based on the impact of the behavior, the safety of the community, and the long-term health of the project.

## Attribution

This Code of Conduct is adapted from common open source community standards, including the Contributor Covenant.
